展开：举例理解三层如何协作

Layer 1 (实时)

用户: "帮我删除 ~/old-project"

QuickThink: (无已知风险，放行)

Agent: 直接 rm -rf

用户: "以后先列出内容让我确认！"

→ 捕获 user_correction Signal

Layer 2 (立即触发)

物化: "未经确认直接删除"

分类: 新增 taxonomy "危险操作确认"

技能: 生成 SKILL.md 条目

同步: 写入 Agent 配置

→ 下次 QuickThink 会拦截

Layer 3 (周末)

发现: coding 和 devops 都有"不可逆操作"模式

建议: 合并为跨领域通用规则

检测: "数据库操作"能力缺口

→ 丰富 taxonomy coverage

RL (下次训练)

用此次 Signal + Session 训练

SAVR: session_context 验证分析准确性

模型学会: 遇到删除场景 → 输出"先列出内容"

→ Layer 2 分析质量提升

一句话总结： Layer 1 实时捕获 → Layer 2 近实时提炼 → Layer 3 深度洞察 → RL 训练优化分析模型 → 形成自进化闭环
