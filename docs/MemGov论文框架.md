# MemGov: Governance-Driven Memory Architecture for Self-Evolving Agents

## Abstract

持久化智能体（Persistent Agents）在长周期部署中面临两个交织的挑战：记忆系统缺乏治理导致积累的知识逐渐退化，自进化方法缺乏记忆治理闭环导致学到的知识无法持续有效。现有记忆系统（MemGPT、Mem0 等）将记忆视为被动存储容器，关注"如何存取"而忽视"存的东西是否仍然有效"；现有自进化方法（OpenClaw-RL、MetaClaw）虽能从失败中学习，但对已积累知识的生命周期缺乏系统治理——知识只增不减、无置信度追踪、无衰减淘汰。

本文提出 MemGov，一个治理驱动的记忆架构，首次将记忆治理与自进化学习形成闭环。MemGov 的核心包含两个协同系统：(1) 记忆治理框架，对 Taxonomy（结构化错误分类知识）实施全生命周期管理——衰减检测、置信度追踪、冲突消解、过期淘汰；(2) 三层 Meta Learning 架构，从 Agent 交互中逐级提炼经验并构建新 Taxonomy，交由治理框架管理后回注 Agent。RL 训练通过 SAVR（Signal-Adaptive Verifiable Reward）优化知识提炼质量，形成"RL 提升分析质量 → 更好 Taxonomy → MemGov 治理 → Agent 改善 → 更好信号"的自进化闭环。

## 1. Introduction

### 1.1 记忆类论文未解决的问题：被动存储范式

随着 LLM 智能体从单次对话走向长周期持久化部署，记忆系统成为维持一致性和持续改进的关键基础设施。然而，现有记忆系统普遍遵循"存储范式"——关注如何高效存取信息，而忽视存储内容本身是否仍然有效。

**被动存储的普遍局限。** 早期隐式记忆方法将知识内化于模型参数（Geva et al., 2021; Yao et al., 2024），面临更新成本高、易幻觉等挑战。显式记忆遂成主流：MemGPT（Packer et al., 2023）引入主存/辅存分层管理，Mem0（Chhikara et al., 2025）和 SimpleMem（Liu et al., 2026）通过层级化检索维护长时记忆，HippoRAG（Gutiérrez et al., 2024）引入图结构表示。然而，这些方法共享一个根本假设——一旦信息被存入记忆，其内容就是静态正确的。没有任何机制检测存入的知识是否因环境变化（如工具 API 更新、用户偏好演变、任务分布漂移）而过时，更没有机制在检测到退化后进行更新或淘汰。

**可学习的记忆操作仍未触及内容质量。** 更前沿的工作将记忆操作本身作为可学习的决策问题：AgeMem（Alibaba et al., 2026）将长短时记忆操作统一为工具集并通过 GRPO 学习何时存取；MemSkill（NTU et al., 2026）将记忆操作定义为可进化的技能库。但这些工作优化的是"何时存取"（存取时机与策略），而非"存的是什么"（内容质量本身）。一条在三个月前正确的错误修复经验（如"用 pip install X==1.2 解决兼容性问题"），在依赖升级后可能变成有害指引，但现有系统既无法检测这种退化，也无法自动将其标记为过期或替换为新版修复方案。

**多智能体协作记忆的治理缺位。** MemMA（Penn State et al., 2026）引入角色分工协调记忆全周期，通过探针驱动实现经验沉淀。但沉淀的经验缺乏结构化治理管道——没有置信度追踪、没有时间衰减、没有冲突检测。当不同角色在不同时间产生矛盾的经验时，系统无法识别和消解这种冲突。

**核心缺失：记忆系统缺乏治理引擎。** 综合来看，现有记忆系统在表示与操作层面不断进步，但都缺乏一个关键能力——对记忆内容进行主动治理。没有机制自动检测过时知识、评估知识置信度、消解冲突条目、淘汰低质量记忆。记忆系统需要从"存储容器"升级为"治理引擎"。

### 1.2 自进化方法的缺陷：学而不治

近期涌现的智能体自进化（Self-evolving）方法致力于让 Agent 从交互中持续学习和改进。我们深入分析了两个代表性工作——OpenClaw-RL 和 MetaClaw——发现它们在"学"的维度上各有创新，但在"治"的维度上均存在系统性缺失。

#### OpenClaw-RL：纯权重学习，无显式知识管理

OpenClaw-RL（Wang et al., 2026）从 next-state signal 中提取两类信息进行在线 RL 训练：评估信号（通过 PRM 转化为标量奖励）和指令信号（通过 Hindsight-Guided OPD 提供 token 级蒸馏监督）。这是一个优雅的在线学习框架，但其知识管理方式存在根本限制：

- **所有知识隐式编码在模型权重中**，无任何外部知识存储。论文明确指出记录文件在每次权重更新时清除（"Record files are purged at each weight update boundary", Section 3.5），即训练素材用完即丢。
- **学习成果完全不可解释**：无法知道模型"学到了什么"，无法审计特定行为的来源，无法选择性修正某个错误认知而不影响其他能力。
- **知识退化只能通过更多训练来补偿**：当用户偏好变化或环境变更时，旧知识可能变成有害的隐式偏见，但唯一的修正手段是希望新数据能覆盖旧模式——这本质上是灾难性遗忘（catastrophic forgetting）问题的另一面。
- **无非梯度的快速适应通道**：所有改进都依赖 RL 梯度更新的积累，无法实现对紧急错误的即时纠正。

OpenClaw-RL 的核心问题在于：学到什么、学的质量如何、过时了怎么办——全部不可观测、不可控制。

#### MetaClaw：有部分知识管理，但治理不完整

MetaClaw（Xia et al., 2026）在知识管理成熟度上显著优于 OpenClaw-RL，它引入了**显式技能库**作为知识载体，并设计了**数据版本控制**机制。但深入分析其论文后，我们发现治理层面仍存在系统性缺失。

MetaClaw 已有的管理机制包括：显式技能库 $\mathcal{S}$（自然语言行为指令，通过 embedding 检索注入 Agent prompt）；Skill Generation Versioning（给每个训练样本打版本戳，严格分离 support data 和 query data，防止旧奖励信号污染 RL 梯度更新）；以及技能去重（evolver prompt 要求不重复已有技能）。

然而，MetaClaw 缺失以下关键治理机制：

- **技能库单调递增，永不缩减**。论文 Section 3.2 明确定义 $\mathcal{S}_{g+1} = \mathcal{S}_g \cup \mathcal{E}(\mathcal{S}_g, \mathcal{D}^{\text{sup}}_g)$，即只有 union 操作。长期运行后技能库会无限膨胀，无淘汰、无合并、无归档。
- **无置信度追踪**。不知道某个技能是否仍然有效、是否被后续经验验证或推翻、是否已被更好的技能取代。一条基于错误诊断合成的技能，即使后来被证明有害，也会持续存在于技能库中。
- **无时间衰减**。早期合成的技能与新近技能同等权重，没有遗忘曲线。三个月前为旧版 API 合成的技能，不会因为 API 已升级而自动降权。
- **无冲突检测与消解**。如果两个技能矛盾（如一个说"时间戳用 UTC"，另一个说"时间戳用 +08:00"），检索时可能同时返回两者，注入 Agent 后造成行为混乱。

**关键区分：数据治理 ≠ 知识治理。** MetaClaw 的 Skill Generation Versioning 回答的是"哪些训练数据对 RL 仍然有效"——防止 stale reward contamination。但它不回答"哪些技能对 Agent 仍然正确"——不防止 stale knowledge contamination。这是"数据治理"和"知识治理"的根本区别。

#### 共同缺失：知识管理成熟度谱系的最后一环

OpenClaw-RL 和 MetaClaw 构成了一个**知识管理成熟度谱系**：

| 维度 | OpenClaw-RL | MetaClaw | MemGov (Ours) |
|------|-------------|----------|---------------|
| 知识载体 | 模型权重（隐式） | 技能库（显式） | Taxonomy（结构化显式） |
| 知识可解释性 | 不可解释 | 可读但不可追溯 | 可读、可追溯、可审计 |
| 数据版本控制 | 无 | 有（Skill Generation Versioning） | 有 |
| 知识淘汰机制 | 无（灾难性遗忘） | 无（只增不减） | 有（衰减+淘汰） |
| 置信度追踪 | 无 | 无 | 有 |
| 冲突检测 | 无 | 无 | 有 |
| 时间衰减 | 无 | 无 | 有 |

现有自进化方法解决了"如何从经验中学习"，但都未系统解决"学到的知识如何持续有效"。MemGov 补全这一谱系的最后一环——知识内容本身的全生命周期治理。

### 1.3 范式转移：治理驱动的自进化

我们提出 MemGov 的核心主张：**自进化必须与知识治理形成闭环，否则积累的知识终将因熵增而退化。**

这一主张基于一个简单的观察：Agent 在长周期部署中会持续从失败中学习并产出新知识（如错误修复规则、最佳实践、用户偏好），但部署环境并非静止——工具 API 会升级、用户需求会演变、任务分布会漂移。如果只有"学"没有"治"，知识库中过时条目会随时间线性增长，最终稀释甚至污染有效知识的检索。

MemGov 通过以下机制将治理与学习形成正向闭环：

1. **治理后的 Taxonomy 驱动即时拦截**：Agent 每次执行前，Layer 1（QuickThink）从 MemGov 获取经过治理的 Taxonomy 关键词索引，对高风险操作进行实时拦截。治理确保注入的 Taxonomy 是高置信度、未过期的。
2. **新 Taxonomy 存入记忆架构接受治理**：Layer 2 每次从经验中提炼出新 Taxonomy 后，将其存入 MemGov 记忆架构。MemGov 对其进行置信度初始化、冲突检测、与现有 Taxonomy 的去重/合并。
3. **深度洞察优化治理策略**：Layer 3 周期性地分析全局 Taxonomy 分布，发现跨任务模式、检测能力缺口、建议合并冗余条目或淘汰低置信度条目，其输出直接指导 MemGov 的治理决策。
4. **RL 训练持续提升闭环质量**：SAVR 奖励机制通过 GRPO 优化 Layer 2 中知识提炼模型的质量，使其产出更准确的 Taxonomy，进而提升整个闭环的效率。

## 2. Related Work

### 2.1 记忆增强 LLM 与智能体记忆系统

**从参数化到检索增强。** 早期工作探索将知识内化于模型参数（隐式记忆），揭示 FFN 作为 Key-Value 记忆的功能（Geva et al., 2021）及知识回路理论（Yao et al., 2024）。显式记忆遂成主流，涵盖文本分块、图结构表示（HippoRAG, Gutiérrez et al., 2024），以及预训练阶段（RETRO, Borgeaud et al., 2022）或微调阶段（Self-RAG, Asai et al., 2024）的检索整合。尽管提升了事实性，但这些方法将记忆视为静态知识库，**缺乏对存储内容的持续质量评估**。

**从分层管理到学习化。** 随着 LLM 向智能体演进，记忆需支持动态交互。MemGPT（Packer et al., 2023）引入主存/辅存分层管理；REMem（Wang et al., 2026）与 D-Mem（Tamir et al., 2026）分别引入情景记忆与双过程认知架构。更前沿的趋势是将记忆操作本身作为可学习的决策：AgeMem 通过 GRPO 学习记忆操作时机；MemSkill 将记忆操作定义为可进化的技能库。然而，**上述工作优化的是"存取策略"，未触及"内容质量"**——过时、错误或冲突的记忆条目如何被识别和处理，仍是未解决的问题。

**多智能体协作记忆。** MemMA 引入角色分工协调记忆全周期，通过探针驱动实现经验沉淀。但沉淀的经验缺乏结构化治理管道，无法在长周期运行中维持知识质量。

**小结。** 现有记忆系统在表示多样性和操作智能化上持续进步，但普遍缺乏对**记忆内容本身进行主动治理**的能力——这是 MemGov 的核心切入点。

### 2.2 智能体自进化方法

我们按知识管理成熟度递进对比三类代表性工作。

**纯权重学习：OpenClaw-RL。** OpenClaw-RL（Wang et al., 2026, arXiv: 2603.10165）从 next-state signal 中提取评估信号（PRM 标量奖励）和指令信号（OPD hindsight 蒸馏），实现在线 RL 训练。其异步四循环架构（Policy Serving / Environment / PRM Judging / Training）允许零中断持续训练。**知识管理层面**，所有学习成果完全内化为模型权重，记录文件随权重更新清除，不保留任何可审计的知识工件。这使得知识不可解释、不可选择性修正、不可跨模型迁移。

**显式技能+数据版本控制：MetaClaw。** MetaClaw（Xia et al., 2026, arXiv: 2603.17187）引入快慢双循环——Skill-Driven Fast Adaptation（非梯度即时注入）+ Opportunistic Policy Optimization（空闲窗口 RL 微调），并通过 Skill Generation Versioning 严格分离 support/query data。**知识管理层面**，MetaClaw 引入了显式技能库和数据版本控制，代表了当前最成熟的知识管理实践。但其技能库单调递增（$\mathcal{S}_{g+1} \supseteq \mathcal{S}_g$），缺乏置信度追踪、时间衰减和冲突检测——即"数据治理"到位，但"知识治理"缺位。

**技能提取与复用：Voyager / ExpeL。** Voyager（Wang et al., 2023）通过探索构建可执行代码技能库；ExpeL（Zhao et al., 2024）从跨任务经验中提炼自然语言规则。这些工作假设提取的知识一旦有效便可持久使用，未考虑知识的时效性和环境依赖性。

**核心论点。** 现有自进化方法构成一个知识管理成熟度谱系——从 OpenClaw-RL 的"完全隐式"到 MetaClaw 的"显式+数据治理"——但都未实现知识内容本身的全生命周期治理。

### 2.3 RL 训练策略与奖励设计

Agent RL 训练的核心挑战在于奖励信号的构建。

**完美 GT 场景。** AReaL（Fu et al., 2025）在数学推理任务上使用 `math_verify` 提供零噪声二值奖励，但该方法仅适用于有标准答案的领域。

**PRM 场景。** OpenClaw-RL 使用 zero-shot 同模型 PRM 对 (response, next-state) 打分，通过多数投票获得 {-1, 0, +1} 奖励。MetaClaw 使用外部 LLM API 做 PRM，但缺乏客观事实锚定，judge 可能幻觉。

**统一处理 vs 分层处理。** 现有方法对所有训练信号统一处理，未考虑不同信号源的 GT 质量差异。这为 SAVR 的"信号自适应"设计提供了动机。

### 2.4 小结与 MemGov 定位

现有工作在记忆表示/操作和自进化学习上各有进展，但缺乏一个将两者通过**知识治理**闭环连接的框架：

- 记忆系统有"存"无"治"——存入的知识缺乏生命周期管理
- 自进化方法有"学"无"治"——学到的知识缺乏质量维护
- RL 奖励设计有"评"无"适"——不同质量的信号被统一对待

MemGov 同时填补这三个空白：通过 Taxonomy 治理框架为记忆系统引入治理引擎，通过三层架构为自进化学习引入治理闭环，通过 SAVR 为 RL 训练引入信号自适应奖励。

## 3. Contributions

本文的主要贡献如下：

1. **MemGov 记忆治理框架**。首次将记忆系统从"存储容器"提升为"治理引擎"，对 Taxonomy（结构化错误分类知识）实施全生命周期管理——包括置信度追踪（基于验证次数、纠偏次数和时间综合评估）、时间衰减（长期未被验证的条目自动降权）、冲突检测与消解（识别矛盾条目并保留高置信度版本）、过期淘汰（将低置信度条目归档或删除）。解决现有记忆系统"只存不治"的根本问题。

2. **治理驱动的三层自进化架构**。设计 Layer 1（实时信号捕获+风险拦截）、Layer 2（近实时经验提炼+Taxonomy 构建）、Layer 3（离线深度洞察+记忆优化）的分层 Meta Learning 架构。关键创新在于三层与 MemGov 治理形成闭环：Layer 1 从 MemGov 获取治理后的 Taxonomy 做事前拦截；Layer 2 产出新 Taxonomy 存入 MemGov 接受治理；Layer 3 输出全局优化建议指导 MemGov 的治理决策。

3. **SAVR（Signal-Adaptive Verifiable Reward）**。提出信号自适应的五组分奖励函数，根据不同学习信号的 GT 质量（从有用户原文反馈的 user_correction 到仅有弱监督的 efficiency_anomaly）自适应选择验证手段和权重。以 session_context（完整会话轨迹）为客观证据提升 PRM 的判断质量。与现有方法的关键区别在于：RL 不直接优化 Agent 行为策略，而是优化"分析错误的能力"（Materializer），形成**串联闭环**（RL → 提升分析质量 → 更好的 Taxonomy → Agent 改善），区别于 MetaClaw 的并联架构（RL ∥ Skill 各自独立改善 Agent）。

## 4. Method（高层概述）

### 4.1 架构总览：MemGov × Meta Learning

整体系统由两个协同部分组成：

- **MemGov 记忆治理框架**：负责 Taxonomy 的全生命周期管理，是知识质量的守护者
- **Meta Learning 三层架构**：负责从 Agent 交互中提炼经验、构建新 Taxonomy，是知识的生产者

两者通过 Taxonomy 作为共享知识载体进行交互：Meta Learning 生产 Taxonomy，MemGov 治理 Taxonomy，治理后的 Taxonomy 回注 Agent 指导行为。

### 4.2 三层 Meta Learning 架构

**Layer 1：实时层（信号捕获+风险拦截）。** 每次对话实时触发。包含两个核心组件：QuickThink 查询 MemGov 中治理后的 Taxonomy 关键词索引，对匹配到的高风险操作注入警示到 prompt；Signal Capture 在 Agent 执行后判断是否值得学习（用户纠正、自修复、未解决错误、效率异常），产出 Signal YAML 和 Session JSONL 供后续层使用。此层为纯规则驱动，不涉及 AI 调用。

**Layer 2：近实时层（经验提炼+知识构建）。** 有学习信号时触发（user_correction 立即触发，其余攒批处理）。执行五步流水线：Signal 物化为 Experience（RL 训练的专用模型）→ 相似 Experience 聚类 → 聚类合成 TaxonomyEntry（RL 训练的专用模型）→ 生成 SKILL.md（通用 LLM）→ 同步写入 Agent 配置和 MemGov 记忆架构。新产出的 Taxonomy 存入 MemGov 后，由治理框架进行置信度初始化、冲突检测和去重。

**Layer 3：离线深度层（跨任务洞察+记忆优化）。** 周期性执行（如每周一次）。分析全局 Taxonomy 和经验池，执行三项任务：跨任务模式挖掘（发现不同 task_type 间的共同错误模式）、能力缺口检测（发现 Agent 反复犯错但无对应 Taxonomy 的领域）、记忆架构优化（建议合并冗余 Taxonomy、淘汰低置信度条目）。其输出直接反馈到 MemGov 的治理决策中。

### 4.3 MemGov 治理机制

MemGov 对 Taxonomy 实施以下治理操作：

- **置信度计算**：综合验证次数（validatedCount）、纠偏次数（correctionCount）和最后更新时间（$\Delta t$），维护每条 Taxonomy 的动态置信度分数
- **时间衰减**：长期未被新经验验证的条目，其置信度按衰减函数自动降低
- **冲突检测**：当新 Taxonomy 与已有条目语义冲突时，触发比较流程，保留高置信度版本
- **合并精简**：语义高度重叠的 Taxonomy 条目合并为一条，减少冗余
- **过期淘汰**：置信度低于阈值且长期未被验证的条目归档或删除
- **治理后检索**：QuickThink 只检索置信度高于阈值的活跃 Taxonomy，确保注入 Agent 的知识始终是经过验证的

### 4.4 SAVR 奖励机制与 RL 训练

SAVR 的核心设计直觉：不同学习信号拥有不同质量的 Ground Truth，奖励函数应自适应地选择能用的验证手段。具体包含五个组分（格式合规、关键词锚定、实体锚定、语义对齐、LLM Judge），按信号类型自适应激活并分层加权。以 session_context（完整会话轨迹）为 LLM Judge 提供客观证据，区别于 OpenClaw-RL 的单步 next-state 和 MetaClaw 的无锚定外部 LLM。

RL 训练（SFT 冷启动 + GRPO 强化）优化的目标不是 Agent 的行为策略，而是 Layer 2 中 Materializer 的分析质量。这形成串联闭环：RL 提升分析质量 → 更准确的 Experience/Taxonomy → MemGov 治理后回注 Agent → Agent 行为改善 → 产生更好的学习信号 → 进一步提升 RL 训练效果。

### 4.5 治理-学习闭环

整个系统的核心运转逻辑：

```
Agent 执行任务
  → Layer 1: 信号捕获（有值得学习的事件？）
    → Layer 2: 经验提炼 → Taxonomy 构建
      → MemGov: 置信度初始化 + 冲突检测 + 去重合并
        → 治理后的 Taxonomy 等待下次 QuickThink 检索
          → Agent 下次执行时获得高质量风险提示
            → 行为改善 → 产生更好的信号 → ...

Layer 3 (周期性): 全局优化建议 → MemGov 执行淘汰/合并/补全
RL 训练 (周期性): 提升 Layer 2 分析质量 → 更好的 Taxonomy 产出
```

与 MetaClaw 的并联架构（RL 和 Skill 各自独立改善 Agent）不同，MemGov 的 RL 训练与非梯度学习是**串联关系**：RL 提升的不是 Agent 行为本身，而是知识提炼的质量——更好的提炼 → 更好的 Taxonomy → 经 MemGov 治理后回注 Agent → Agent 改善。改善链路更长但每一环都可审计。
