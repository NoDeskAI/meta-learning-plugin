Step D

组内标准化

Ai = (Ri - mean) / (std + ε)
Step E

裁剪 Loss 更新

ε_clip = 0.2

β_kl = 0.01

GRPO 核心公式（与 OpenClaw-RL / MetaClaw 一致）:

L = -E[ min( r(θ)·A, clip(r(θ), 1-ε, 1+ε)·A ) ] + βkl · KL(πθ || πref)

r(θ) = π_θ(y|x) / π_old(y|x)（importance ratio）

On-policy 循环：生成 → 打分 → 排名 → 更新策略 → 生成更好的候选 → ...

奖励函数：SAVR 五组分

Signal-Adaptive Verifiable Reward — 按信号类型自适应选择验证手段

Rtotal = wsignal × Σ Ractive_components
F

Rformat

[-1.0, +0.2]

格式合规 — 门槛奖励（RLVR 风格）

JSON 解析失败 → -1.0

每个非空字段 → +0.04 (5字段=0.20)

task_type 非法 → -0.05

类型: rule-based | 所有信号类型激活

K

Rkeyword

[0, +0.15]

关键词锚定 — resolution ↔ user_feedback 关键词重叠

keyword_overlap(resolution, user_feedback) × 0.15

好: resolution 含"ls"、"确认"→ 0.12

差: resolution 无关键词 → 0.00

类型: rule-based | 仅 user_correction

新增
E

Rentity

[0, +0.30]

实体锚定 — root_cause 是否引用 session 中的具体实体

实体引用率 (命令/路径/错误码) × 0.15
步骤锚定 ("执行 X 时"、"第 N 步") × 0.08
因果链 ("因为"、"导致") × 0.07

好: "执行 rm -rf 时未先 ls 列出内容" → 0.25

差: "操作流程不规范" → 0.02

类型: rule-based | error 类信号激活

A

Ralign

[-0.1, +0.35]

Resolution 语义对齐 — 与 user_feedback 的 embedding 相似度

cosine_sim(embed(resolution), embed(user_feedback)) × 0.35

embedding: bge-small-zh

类型: embedding | 仅 user_correction

核心创新
P

Rprm — LLM-as-Judge with Session Context

[-0.20, +0.20]

以 session_context（完整会话轨迹）为客观证据，判断分析是否准确

机制

LLM 评审员 × 3 次独立采样 → 多数投票

输出 {-1, 0, +1} × 0.20 权重

三个变体

prm: 评估整体准确性 (user_correction, self_recovery)

prm_diag: 仅评估诊断 (unresolved_error)

prm_weak: 弱约束 (new_tool, efficiency)

与 OpenClaw-RL 的类比

OpenClaw-RL: 评估 (回复, next-state) → "回复是否有帮助？"
我们: 评估 (分析, session_context) → "分析是否准确？"

我们的证据更丰富 — 完整会话轨迹 vs 单条 next-state

类型: LLM judge | 按信号类型选择变体

信号类型 → 激活组分 + 权重对照表

信号类型 F K E A P 权重 GT 来源
user_correction ✓ ✓ ✓ ✓ prm 1.0 user_feedback + session_context
self_recovery ✓ — ✓ — prm 0.8 session_context（完整轨迹验证）
unresolved_error ✓ — ✓ — prm_diag 0.5 error_snapshot + session_context
new_tool ✓ — — — prm_weak 0.25 无硬 GT
efficiency_anomaly ✓ — — — prm_weak 0.2 无硬 GT

设计逻辑：GT 质量越高的信号 → 激活更多组分 + 更高权重 → 贡献更大训练梯度
