# DeskClaw Meta-Learning SOP: 自进化记忆与逻辑提炼

> **版本：** 1.0 (2026-03-09)
> **状态：** 待实施（逻辑固化）
> **目标：** 实现从“被动记录”到“主动进化”的跃迁，通过 Action-Think-Refine 循环持续优化 Agent 性能。

---

## 1. 核心逻辑架构 (Action-Think-Refine)

本 SOP 参考论文 *Evo-Memory (arXiv:2511.20857)*，将任务处理流程重构为以下三个阶段的闭环。

### A. Think (前置思考与历史对齐)
在接收到任务请求后，不立即执行，先进行历史对齐：
1.  **经验检索**：搜索 `memory/` 和 `MEMORY.md`，寻找与当前任务关键词相似的历史记录。
2.  **避坑分析**：重点识别历史记录中的“失败反馈”或“纠错记录”。
3.  **策略初始化**：根据历史经验，形成第一版执行 SOP。

### B. Act (精准执行与反馈捕获)
在执行过程中，实时监控环境反馈：
1.  **执行操作**：按照 Think 阶段形成的策略进行工具调用。
2.  **异常捕获**：若出现报错（如 400 Bad Request、502 等），立即停止，进入临时 Refine 环节分析原因（如通过 `cat` 源码或 `web_search` 查文档）。
3.  **用户反馈**：记录用户对执行结果的正向或负向评价。

### C. Refine (记忆提炼与逻辑固化)
这是“进化”发生的核心阶段。任务结束后（无论成功失败），必须执行以下动作：
1.  **剪枝 (Pruning)**：从本次对话中剔除无效的废话、重复的报错尝试。
2.  **碎片 Case 提取**：将成功的调用路径、正确的参数格式抽象为“代码/技能卡片”。
3.  **逻辑提炼 (Logic Refine)**：
    *   **失败提炼**：如果是从失败中修复的，总结“报错特征 -> 根本原因 -> 修复方案”。
    *   **规则提炼**：如果是重复性任务，总结“普适 SOP -> 避雷指南”。
4.  **持久化写入**：将提炼后的内容以 `[META-Learning]` 为前缀写入当天的 `memory/YYYY-MM-DD.md`，若同一类提炼出现 3 次以上，由 Agent 自主提议合并入 `MEMORY.md`。

## 2. 知识入库标准（META 学习卡片模板）

提炼后的知识应遵循以下 Markdown 格式，以便于下次检索：

```markdown
### [META-Learning] <知识/技能名称>
*   **场景描述**：在什么任务下，遇到了什么问题。
*   **避坑指南**：容易犯错的点（剪枝后的结论）。
*   **标准 SOP**：正确的执行逻辑或调用命令格式。
*   **提炼逻辑**：为什么这样有效（本质原因）。
```

---

## 3. 对未来自身的指令（自修改指引）

当你（未来的灵敏）被要求“启用 Meta-Learning 模式”时，你应该修改你的系统指令（System Prompt）或 `AGENTS.md` 中的“行为纪律”部分，加入以下约束：

1.  **强制 Refine 步**：在每个任务完成的输出之后，自动在后台（或 memory 中）追加一次 Refine 思考。
2.  **优先查源码**：面对自研或文档模糊的工具，在尝试失败 1 次后，强制执行“源码阅读”动作。
3.  **记忆分级管理**：将 `MEMORY.md` 视为“规则库（Rules）”，将 `memory/*.md` 视为“经验池（Experiences）”，并在执行前强制进行跨库对齐。

## 4. 实施备注
*   **触发器**：复杂代码任务、新技能调用、用户纠错行为。
*   **性能指标**：步骤效率（Step Efficiency）提升 20%+，Token 消耗率降低。
